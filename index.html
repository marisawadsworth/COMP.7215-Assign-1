Client-Server Computing


What is client-server computing?

Client-Server computing is a model architecture on the web that splits computers into two sections—clients and servers. Computers that ask for and request services are called clients, and computers that serve clients are called servers.


Clients can be a machine or program, like a laptop, mobile, desktop etc.
A client program is a program that allows the user to make a request, such as a web browser, word processor, or photo editing software.


Servers are powerful computers dedicated to managing disk drivers (file servers), printers (print servers), or network traffic (network servers).
A single server can serve multiple clients at the same time, and run multiple servers on one single machine. They are called virtual servers. “Apache – Serves HTTP request, and Database Servers runs a database management system such as My SQL and other types of servers.”[9]


Uses of client-server computing

The web is a service which is built on top of the internet to allow computers to share and exchange data smoothly and reliably.
Data includes images, videos, documents etc. in any form and for that sometimes the web is referred to as a client-server model because there are always computers that ask for data which are called clients and the computers that give the data are servers.

Why has client-server evolved

“Client-server computing used to be just a mainframe connecting to dumb terminals.”[9]
Since then, technology has evolved. The capacity of personal computers was largely ignored, many companies started to think about sharing, or splitting, any of the processing demands between the client, and the mainframe.
Client-server technology evolved out of this for better computing control and more computing value. It is a way for which software components interact to form a system that can be for multiple users.[1]


Advantages and Disadvantages

Clients on a client-server architecture can be thin or fat. Thin clients are a computer program or application that depends on another computer (server). The client is responsible for running the presentation software.
A fat client is a computer or network that provides the functionality to a central server. E.g. STMP, FTP, DNS server. This runs some or all of the application logic.[3]


Advantages and disadvantages

Thin advantages, thick disadvantages:

  1. Reduces hardware and software maintenance.
  2. Lesser security vulnerability.

Thick advantages, thin disadvantages:

  1. Better data and program processing.
  2. Server performance efficiency.
